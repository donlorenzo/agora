Agora - Developer Documentation

Introduction

High-Level Design

Agora's main component is called the core. It interacts with the outside world through several API interfaces:
  • UI API
  • Plugin API
  • Configuration API
  • File Access API
In addition agora defines a binary protocol with which it communicates with other agora instances.

The UI API

The UI API consists of two parts
  • Core UI API
  • Client UI API
The Agora core implements the Core UI API and injects itself into the client which in turn must implement the Client UI API.

The Core UI API

The Core UI API exposes Agora's administrative functionality to the Client UI Adapter.

The Client UI API

The Client UI API must be implemented by Client UI Adapters to be able to be loaded by the Agora Core. It provides functionality to inform the UI about changes in the core.

The Plugin API

The Plugin API consists of two parts
  • Core Plugin API
  • Client Plugin API
The Agora core implements the Core Plugin API and injects itself into the client which in turn must implement the Client Plugin API.

The Core Plugin API

Agora exposes certain functionality to Plugins through the Core Plugin API. This includes
  • opening a connection to a remote Agora node
  • sending/receiving data to/from a remote Agora node
  • adding Plugin specific controls to the UI and keeping it up to date
